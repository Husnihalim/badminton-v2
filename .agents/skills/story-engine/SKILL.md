# KelabSukan Story Moment Detection & Generation Rules

This skill defines the rules for the KelabSukan Story Engine. It governs how match, participant, and session data are analyzed to generate compelling stories, highlights, and WhatsApp-ready social share cards.

## When to Apply

Reference these guidelines when:
- Designing or implementing story generation logic
- Creating or editing moment UI cards on the dashboard
- Writing WhatsApp/social share text generators
- Editing or modifying [storyMoments.ts](file:///Users/abc/Documents/Badminton%20v2/src/lib/storyMoments.ts)

---

## 1. The Story Object Schema

All stories generated by the engine must match or extend the computed story type shape. Avoid arbitrary text fields; keep data structured and referenceable back to the source objects (matches, clubs, profiles).

```typescript
export type StoryMoment = {
  id: string;
  type: string; // e.g., 'close_match', 'comeback', 'nemesis'
  scope: 'player' | 'club' | 'rivalry' | 'pair' | 'session';
  title: string;
  summary: string;
  proof: string; // e.g., "Won 21-19, 18-21, 22-20" or "5-0 H2H"
  clubId: string;
  eventId?: string;
  matchId?: string;
  playerIds: string[];
  mediaIds?: string[];
  shareText: string; // WhatsApp optimized text
  createdAt: string;
};
```

## 2. Standard Story Type Rules

The story engine should detect and generate moments deterministically based on match results and head-to-head records:

| Story Type | Trigger Condition | Example Headline |
|---|---|---|
| **Close Match** | Third game difference is $\le$ 2 points (e.g., 22-20, 21-19). | "Edge of the Seat Finish" |
| **Comeback Win** | Player/pair loses Game 1 but wins Game 2 and Game 3. | "The Great Escape" |
| **Clean Sweep** | A player/pair wins all matches in a session without losing a single game. | "Flawless Night" |
| **Win Streak** | Player/pair wins 3 or more consecutive matches. | "On Fire: [Name] Wins 5 in a Row!" |
| **Nemesis H2H** | Head-to-head record between two players is skewed (e.g., 5-0, 6-1). | "The Curse Continues" |
| **Unbeaten Pair** | Double partnership has a 4+ match winning streak together. | "Unbreakable Duo" |
| **Nemesis Broken** | Player defeats an opponent they had lost to in their last 3+ consecutive matches. | "Giant Slayer: [Name] Breaks the Curse" |

## 3. WhatsApp Share Formatting Rules

WhatsApp is our primary distribution layer. Shared captions must be compact, readable on a phone screen, and format-rich.

### Share Formatting Template
```text
🔥 *[Headline]*
🏆 [Summary]

📊 *The Proof:* [Proof Detail]
📍 [Club Name] | [Date]

Read the full match report & view stats:
🔗 [Public Proof URL]
```

### Formatting Checklist:
- **Bold tags**: Wrap titles and sections in asterisks (`*Section Name*`).
- **Emojis**: Use contextual sports emojis (🏸, 🎾, 🔥, 🏆, 📊, ⚡). Keep it to a maximum of 3-4 emojis per message to avoid spammy text.
- **Link Integrity**: Every share message must end with a public proof link. No links requiring authentication.

## 4. Banter & Safety Guardrails

To keep the platform safe, inclusive, and fun, the story engine and AI rewrite components must adhere to strict content rules:

> [!IMPORTANT]
> **Banter Boundaries**
> - **Allowed**: Sport-based, data-driven banter (e.g., "nemesis", "rivalry", "winless streak against X").
> - **Forbidden**: Body or appearance jokes, private-life gossip, age-related comments, political or religious references, and mean-spirited personal attacks.
> - **Youth Privacy**: If a player is registered as a minor, redact their profile photo and replace their display name with initials in public feeds.
