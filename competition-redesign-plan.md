# Competition Redesign Plan

## Overview
Simplify competition management into an intuitive linear flow for club admins. Replace the current system (friendlies/tournaments/pools/playoffs/brackets) with just two competition types: **Friendly** (2 clubs) and **League / Round-robin** (3+ clubs).

---

## Notification & Announcement System

Two channels:
- **🔔 User notification** (`notifications` table) — direct to individual user's bell icon
- **📢 Club notice** (`club_messages` table or `club_activity` feed) — visible to all club members on the club page

### Trigger Map

| Event | Channel | Recipient | Message |
|-------|---------|-----------|---------|
| **Competition created** | 🔔 Notification | All invited clubs' admins | `"{HostClub} invited you to {compTitle}"` |
| | 📢 Club notice | Host club members | `"New {Friendly/League} created with {ClubB, ClubC...}"` |
| **Club confirms lineup** | 🔔 Notification | Competition creator + other club admins | `"{ClubB} has confirmed their lineup"` |
| **All clubs confirmed** | 🔔 Notification | All club admins | `"All clubs are ready! Set up matchups now"` |
| **Competition started (live)** | 🔔 Notification | All selected players | `"{compTitle} is live! Your match is coming up"` |
| | 📢 Club notice | All club members | `"{compTitle} has started at {venue}"` |
| **Rubber completed** | 🔔 Notification | Both pair participants | `"You {won/lost} {score}"` |
| | 📢 Club notice (optional) | Club members | `"{PairName} beat {OpponentPair} ({score})"` |
| **Competition completed** | 🔔 Notification | All selected players | `"{compTitle} completed! {WinnerClub} wins"` |
| | 📢 Club notice | All club members | `"🏆 {WinnerClub} won {compTitle}!"` |

### Roster Notification Triggers

| Event | Channel | Recipient | Message |
|-------|---------|-----------|---------|
| **Admin picks player** | 🔔 Notification | Selected player | `"{Club} invited you to play in {compTitle}"` |
| **Open for volunteers** | 🔔 Notification | All club members | `"{compTitle} needs players! Volunteer now"` |
| **Player accepts** | 🔔 Notification | Club admin | `"{PlayerName} accepted the invite"` |
| **Player declines** | 🔔 Notification | Club admin | `"{PlayerName} declined — replace them?"` |
| **Lineup confirmed** | 🔔 Notification | All selected players | `"You're in! {compTitle} lineup is set"` |

### Player-Facing Notifications (Dashboard)

When a player opens `/my-court`, they see:
- Pending roster invites (accept/decline buttons)
- Upcoming competitions they're in
- Live competitions in progress

## User Flow

```
1. Create → pick type + club(s) + format + date/venue + roster mode
2. Rostering → each club admin manages players & pairs
3. Declare → each club confirms lineup when ready
4. Matchups → auto-generated by rank, editable pre-game
5. Live → score matchups in order
6. Results → standings & winners
```

---

## 1. Competition Types

| Type | Clubs | Format |
|------|-------|--------|
| Friendly | 2 clubs | Head-to-head, X pairs |
| League | 3+ clubs | Round-robin (every club plays every other club) |

---

## 2. Create Wizard (4 steps)

**Step 1: What kind?**
- Friendly (2 clubs)
- League (3+ clubs)

**Step 2: Who's playing?**
- Friendly → pick 1 opponent club
- League → pick 2+ clubs

**Step 3: Format settings**
- Pairs: 1–7 (flexible, default 5)
- Sets per match: 1 or 3
- Points per set: 15 / 21 / 31
- Date
- Venue
- Roster mode: Admin picks vs Open to members

**Step 4: Confirm → Created (clubs notified)**

---

## 3. Rostering (per club)

**Admin picks mode:**
- Admin selects members from club list
- Selected members get notified, can accept/decline
- Declined → admin gets notification with "Replace" link
- Admin drags available players into pairs (@dnd-kit)
- Admin ranks pairs 1–X (drag to reorder, strongest first)

**Open to members mode:**
- Announcement sent to club members
- Members volunteer (accept)
- Admin sees volunteer pool, drags into pairs, ranks them

**Both modes:**
- Event day: admin can add latecomers, re-pair, re-rank
- Must click "Confirm Lineup" when ready
- Status shown per club: Waiting / Confirmed ✅

---

## 4. Matchups (pre-game)

- Auto-generated when ALL clubs confirm: Pair 1 vs Pair 1, Pair 2 vs Pair 2, etc.
- Any club admin can swap/reorder their pairs before game starts
- "Start Competition" button locks matchups → status becomes Live

---

## 5. Live Scoring

- Matchups play in order (Match 1 → Match N)
- Non-current matchups show "Up Next" or "Completed"
- Uses existing BwfMatchupCard + ScoreRecordingModal

---

## 6. Results

**Friendly:** Winner club + rubber score
**League:** Standings table (Rank, Club, Played, Won, Lost, Rubbers W-L)

---

## Status Flow

`draft → registration → matchmaking → live → completed | cancelled`

## Database Changes

### competitions table (modify)
- Remove `opponent_club_id`, `opponent_club_name`
- Rename `pair_count` → `pairs_count` (remove CHECK 4|5)
- Add: `sets_count` (INT), `points_per_set` (INT), `location` (TEXT), `roster_mode` (TEXT)
- Change `format` values to `friendly` | `league`
- Simplify `status` to: draft | registration | matchmaking | live | completed | cancelled

### NEW: competition_clubs table
- competition_id, club_id, status (invited|confirmed|declined), lineup_confirmed, lineup_confirmed_at

### competition_participants (modify)
- Add `rank` INTEGER (position in club's lineup)

### competition_matchups (modify)
- Add `club_a_id`, `club_b_id`, `winner_club_id`, `round_index`, `locked` BOOLEAN
- Remove pool/bracket complexity
