# Homepage Redesign Proposal — ESPN-Style Grassroots Sports Network

> Goal: Turn kelabsukan.com from a "feature tour" landing page into a live, ESPN-style **sports newsroom** that makes visitors *feel the community* and want to join — without a hard sell.

---

## 1. The Shift in Thinking

| Current homepage | Proposed homepage |
|---|---|
| "Here are our features — try them" | "Here is the sport happening right now — join in" |
| Product widgets (playground, mockup generator) | Live feed of real drama, scores, stories, people |
| Static hero with stock rackets | Live ticker + hero story of the week |
| Generic CTA "Start Your Club free" | Social-proof CTAs rooted in real nearby activity |
| Clubs buried at the bottom | Clubs, members, and marketplace surfaced by **location** |

The pitch to a visitor is no longer "use our tool" — it's **"your scene already exists here, come find it."**

---

## 2. Page Structure (top to bottom)

### A. Live Score Ticker Bar (sticky, top of page)
Thin ESPN-style horizontal scroller, always visible on scroll.
- Rotating: latest match results across all public clubs → "LEP BC def. Smashers PJ 21–19, 17–21, 22–20 · 3h ago"
- "🔥 Coach Fai's 12-win streak ended tonight"
- Live count: **"247 matches recorded this week across 38 clubs"**
- A single "Find clubs near you" pin icon → jumps to location search.

Purpose: instant proof that the platform is *alive*.

### B. Lead Story Hero
One big editorial card — the "story of the week", auto-generated by the story-engine (comeback, streak, nemesis, upset).
- Full-bleed image (club action shot or player card art).
- Headline (ESPN tone): `"THREE MATCH POINTS DOWN. Fai REFUSED TO DIE."`
- Subhead: club name, sport, location chip (e.g. · Petaling Jaya · Badminton).
- Two CTAs: **"Read the moment"** → story page; **"Join this club"** → if club has open approval.
- Below: 3 secondary story thumbnails ("More from the arena").

This replaces the current static racket collage hero.

### C. "Find Your Scene" — Location Search Module
The single most important new module. Prominent, above the fold-ish.

```
[ 📍 Use my location ]  or  [ ____ Enter your city/postcode ____ ]   [ Search clubs ▶ ]
```

Returns three side-by-side result columns, **all scoped to the visitor's location**:
1. **Clubs near you** — cards with member count, sport, next session, "Open / Invite-only" badge.
2. **Marketplace near you** — gear, coaching, court bookings within radius.
3. **Players near you** — public player cards (with consent), Elo, looking-for-game status.

Default radius: 25 km (adjustable slider 5 / 25 / 100 km). If no location set → show "Trending near [country]" as fallback so the page is never empty.

This directly answers your requirement: *each club has a home base location, marketplace is location-based.*

### D. The Desk — Live Tabs (kept & sharpened)
Reuse the existing tabbed "Sports Desk" idea but make it newsroom-grade:
- **🔥 Stories** — auto-generated story moments feed, infinite scroll.
- **🏆 Leaderboards** — toggle between public clubs; show top 10 + ranking movement arrows (▲3 ▼1).
- **💀 Drama** — rivalries, streak-breakers, nemesis pairs, upsets. Tagged & dated.
- **⚡ Players** — featured public player cards, "athlete of the week", rising Elo.

Each item links deeper — never a dead-end card. Clicking a story → public story page; clicking a player → public player card; clicking a club → club homepage (which already exists at `/club/:clubId`).

### E. Featured Clubs Wall
Masonry grid of **publicly featured clubs**, each card showing:
- Club crest + accent color strip.
- Name, sport, **location chip**, member count.
- Latest headline from that club ("Just dethroned the reigning champ").
- "Visit club" → `/club/:clubId`.

This is your requirement: *more clubs, more members publicly featured on homepage = intrigue to join*. Clubs opt in to be featured (admin toggle in club settings). Rotates daily so it doesn't feel curated-bias.

### F. Faces of the Community
Horizontal scroller of **public player cards** with consent flag turned on.
- Photo/avatar, name, club, Elo, signature stat ("12-0 this season"), location chip.
- "This athlete plays 8km from you" — location-aware.

Real humans > abstract features. This is the softest, strongest "you should join" pitch there is.

### G. Wins & Heartbreaks — The Emotional Counterweight
You asked for sad stories, good stories, drama. A two-column editorial section:
- **🏆 Wins** — comeback of the month, new record, first-time champion, veterans returning.
- **💔 Heartbreaks** — the streak that ended, the retired captain, the relegation battle lost.

Story-engine already supports deterministic moment generation; this section exposes that output publicly and lets users share to WhatsApp with the existing `StoryShareGroup`.

### H. The Numbers (social proof band, no CTA)
Animated counters — factual, not boastful.
- Active clubs · Active athletes · Matches recorded · Stories shared · Cities covered.

No "join now" button here. The point is the number itself does the selling.

### I. Marketplace Preview (location-aware)
A compact "deal of the week near you" rail:
- Used rackets, coaching slots, court rentals — each with distance chip ("2.4 km").
- Links into the existing marketplace flow but filtered by visitor location.

### J. Footer CTA — Quiet, Not Screaming
Just one line: *"Your club, your scene, your story. Find it at kelabsukan.com."* + **Find clubs near me** button. No giant banner.

## 3. Data / Schema Implications

To support this homepage we need three things on the database side:

### 3.1 Club home-base location (new columns on `clubs`)
```sql
alter table clubs
  add column home_address text,
  add column home_city text,
  add column home_postcode text,
  add column home_lat double precision,
  add column home_lng double precision,
  add column home_radius_km int default 25;

-- Or use PostGIS for proper geo:
alter table clubs add column home_point geography(point, 4326);
create index clubs_home_point_idx on clubs using gist(home_point);
```
Every club must complete a home location at creation (force onboarding step). Publicly shown as city + postcode chip ("Petaling Jaya · 47301"). Exact lat/lng never exposed to strangers — only used for distance ranking.

### 3.2 Marketplace location
The existing marketplace items need the same geo columns + a `pickup_radius_km`. Search uses `ST_DWithin` so buyers only see what's reachable.

### 3.3 Public-featuring flags (consent-driven)
```sql
alter table clubs add column featured_public boolean default false;
alter table profiles add column featured_public boolean default false;  -- player-card consent
```
Existing club settings already has pinning UI — extend it with a "Feature on kelabsukan.com homepage" toggle so admins opt in. Same for players (privacy-skill compliant: never auto-feature minors).

### 3.4 Homepage RPCs (performance skill — avoids api.ts monolith)
One Postgres RPC returning the whole homepage payload in a single round-trip:
```sql
-- rpc.get_homepage_feed(p_lat, p_lng, p_radius_km)
-- returns: lead_story, ticker[], clubs_near[], marketplace_near[],
--          players_near[], featured_clubs[], featured_players[],
--          wins[], heartbreaks[], counters{}
```
Cached / stale-while-revalidate via React Query. No N+1 calls from the client.

## 4. Design Notes (per design-rules skill)

- Keep the arena-lime accent only for **live/active** elements (ticker, "LIVE" chips, CTA primary). Don't paint big hero blocks lime — it cheapens the cue.
- Hero story uses the surfaced card with a dark photo overlay so white headline pops — mirrors ESPN's "Lead L" composition.
- Cards keep `rounded-2xl` + `arena-border` + `arena-surface` tokens; no new shadow system.
- Ticker is a thin horizontal scroller with `scroll-snap` — works mobile-first (the majority of grassroots traffic).
- All sections ship in mobile-stack and desktop-grid variants; no horizontal-only sections (delivery-and-qa skill requires both viewports).

## 5. What Gets Removed / Demoted

The current page has grown into a product-tour artifact. To make room for the newsroom:
- **Record Performance Playground** → move to `/playground` route. Useful demo, wrong page to anchor the homepage.
- **Try Club Setup Widget** → demote into the footer / `/get-started`. The hero should not be a form.
- **Mockup Generator** → already at `/player-home-preview`; remove from landing.

This is a hard cut, but it's the only way the homepage stops looking like a SaaS pitch deck.

## 7. Why This Works (the "without trying too hard" part)

It's not selling software — it's **showing a community in motion**. The visitor's reaction goes:

1. *"Oh, there are 38 clubs near me playing this week."* (ticker)
2. *"That comeback story is wild — what club is that?"* (hero)
3. *"Wait, there's a club 6km from my house with open slots."* (location search)
4. *"Those players look like normal people, not pros — I could do this."* (faces)
5. *"247 matches recorded this week — this is actually happening here."* (counters)

By the time the CTA appears in the footer, the visitor has already convinced themselves. That's the ESPN trick, and it's exactly the spirit of grassroots sport.

## 8. Decisions Confirmed

1. **Distance default** → 25 km, with 5 / 25 / 100 km slider. No country-fallback; "Trending near [geo]" placeholder if location unset.
2. **Lead hero story** → auto-selected from story-engine (comeback / streak / nemesis / upset). Clubs can boost by pinning — no manual editor override.
3. **Featured Clubs Wall** → algorithm-rotated daily based on activity/recency. Clubs opt-in via existing settings toggle (extends `pinned_announcement` pattern). No curated/sponsorship slot yet.
4. **Marketplace preview** → 3-deal rail ("deal of the week near you") with distance chip, links into marketplace. No filters on homepage.
5. **Minors policy** → feature on public homepage **only with both club-admin consent AND guardian flag set**. Build `featured_public` on profile + `guardian_feature_consent` boolean. Default off. Privacy skill must review before any minor card ships.
